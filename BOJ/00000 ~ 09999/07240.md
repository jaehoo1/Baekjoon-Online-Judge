# [7240 : Rogutės](https://www.acmicpc.net/problem/7240)
https://www.acmicpc.net/problem/7240

## 문제
드워프는 썰매 언덕을 내려가는 것을 좋아합니다. 내리막길을 내려갈 준비가 되었을 때(처음 순간) 썰매의 속도는 0입니다. 내리막길은 N개의 구간으로 나뉘며, 각 구간마다 썰매의 속도가 어떻게 변하는지를 알 수 있습니다: ai = m/s의 증가 또는 감소.

드워프 썰매는 똑똑합니다. 속도를 측정하고 필요한 경우 속도를 늦춥니다. 구간이 교차하는 지점에서 썰매의 속도가 S m/s를 초과하면 썰매의 속도가 1m/s씩 감소합니다. 썰매는 코스의 끝이 아닌 구간이 교차하는 지점에서만 속도가 변경됩니다.

언덕 아래(트랙의 마지막 구간 이후)에서 썰매가 스케이트를 탈 속도를 출력하세요.

<br/>

**입력**  
첫 번째 줄은 트랙의 구간 수 N과 속도 S를 나타냅니다.

다음 N 행에는 트랙의 해당 구간을 완료한 후 썰매의 속도가 얼마나 변하는지를 나타내는 숫자 ai가 하나씩 들어 있습니다.

<br/>

**출력**  
트랙이 끝날 때 썰매의 속도를 출력합니다.

초기 데이터는 항상 썰매가 전체 트랙을 확실히 완주한다는 것, 즉 속도가 0으로 감소하지 않는다는 것입니다(트랙의 끝을 제외하고).

<br/>

번역 출처 : https://www.acmicpc.net/board/view/140553

## 아이디어
Simulation(Implementation), Arithmetic(Math)

(마지막을 제외하고) 속력 변화를 반영한다.
```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

StringTokenizer st = new StringTokenizer(br.readLine());
int N = Integer.parseInt(st.nextToken());
int S = Integer.parseInt(st.nextToken());
int speed = 0;
for (int i = 0; i < N; i++) {
    int a = Integer.parseInt(br.readLine());
    speed += a;
    if (i < N - 1 && speed > S) {
        speed--;
    }
}
System.out.println(speed);
```

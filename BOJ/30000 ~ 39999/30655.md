# [30655 : Pique Esconde](https://www.acmicpc.net/problem/30655)
https://www.acmicpc.net/problem/30655

## 문제
**문제**  
컴퓨터를 좋아하는 João는 자신의 일상생활에서 하는 모든 일을 자동화하는 프로그램을 어떻게 만들 수 있을지 고민하고 있습니다. Maria는 그들이 숨바꼭질을 할 때 사용할 수 있는 유용한 프로그램을 만들어보라고 그에게 도전했습니다.

João는 아주 좋은 프로그램 아이디어를 생각해냈지만, 아직 컴퓨터 수업을 처음 듣고 있는 단계라서 이 아이디어를 어떻게 구현해야 할지 모릅니다. 따라서, 여러분이 그를 도와주어야 합니다.

João의 아이디어는 숨바꼭질에 참여하는 아이들에게 1부터 n까지 번호를 매기는 것입니다. 프로그램은 입력으로 전체 아이들의 수와 술래가 된 아이의 번호를 받습니다. 그리고 나서, 프로그램은 이미 발견된 n-2명의 아이들의 번호 목록을 받고 아직 찾지 못한 아이가 누구인지 답해야 합니다.

여러분의 임무는 이 프로그램을 구현하여 João가 Maria에게 컴퓨터가 가장 단순한 작업에서도 유용할 수 있다는 것을 보여주도록 돕는 것입니다.

<br/>

**입력**  
입력은 여러 개의 테스트 케이스로 구성됩니다. 각 테스트 케이스의 첫 번째 줄에는 아이들의 수 n과 술래인 아이의 번호 m이 주어집니다(2 ≤ n ≤ 1000, 1 ≤ m ≤ n). 다음 n-2개의 줄에는 각각 이미 발견된 아이들을 나타내는 정수가 하나씩 있습니다.

입력은 n = m = 0일 때 끝납니다. 이 줄은 처리하지 않아야 합니다.

<br/>

**출력**  
각 테스트 케이스에 대해, 프로그램은 아직 발견되지 않은 아이의 번호를 포함하는 한 줄을 출력해야 합니다.

<br/>

번역 출처 : https://www.acmicpc.net/board/view/152082

## 아이디어
Brute Force, Implementation

숨바꼭질에 참여한 사람들(1 ~ n) 중 찾은 사람들을 표시할 배열을 만들어 표시하고 발견 못한 사람을 찾으면 된다.
```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));

while (true) {
    StringTokenizer st = new StringTokenizer(br.readLine());
    int n = Integer.parseInt(st.nextToken());
    int m = Integer.parseInt(st.nextToken());
    if (n == 0 && m == 0) {
        break;
    }
    boolean[] found = new boolean[n + 1];
    found[m] = true;    // 술래도 찾았다고 표시
    for (int i = 0; i < n - 2; i++) {   // 찾은사람 표시
        found[Integer.parseInt(br.readLine())] = true;
    }
    for (int i = 1; i <= n; i++) {      // 못찾은 사람은 찾은사람 배열을 순차탐색하면 찾을 수 있음
        if (!found[i]) {
            bw.write(i + "\n");
        }
    }
}
```

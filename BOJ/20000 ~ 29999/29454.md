# [29454 : Выражение](https://www.acmicpc.net/problem/29454)
https://www.acmicpc.net/problem/29454

## 문제
**문제**  
탁자 위에 $N$ 장의 카드가 놓여있다. 각 카드에는 $-100$ 이상 $100$ 이하의 정수 중 하나가 적혀 있다. $i$번 카드에 적힌 정수를 $A_i$라 하자.

우리는 이 중 **신비한 카드**를 찾으려고 한다. 아래의 조건을 만족하는 카드는 신비한 카드이다.
- 자기 자신에게 적힌 수와 자기 자신을 제외한 나머지 카드에 적힌 수의 합이 같다.

카드의 목록이 주어지면, 몇 번 카드가 신비한 카드인지 구하는 프로그램을 작성해 보자. 단, 주어지는 카드의 목록에는 최소 한 장의 신비한 카드가 있으며, 조건을 만족하는 신비한 카드가 여러 장인 경우에는 그 중 가장 작은 번호를 출력하자.

<br/>

**입력**  
첫째 줄에 $N$이 주어진다. $(2 \le N \le 100)$

둘째 줄에 $A_1, A_2, A_3, ..., A_N$ 이 공백으로 구분되어 주어진다. ($-100 \le A_i \le 100$; $A_i$는 정수)

<br/>

**출력**  
첫째 줄에 문제에서 요구한 신비한 카드의 번호를 출력한다.

<br/>

번역 출처 : https://www.acmicpc.net/board/view/125055

## 아이디어
Brute Force, Implementation, Arithmetic(Math)

수를 입력받으면서 합을 구해놓고 순차 탐색으로 (합에서) 해당 수를 뺐을 때 해당 수와 결과가 같은 값을 찾는다.
```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

int n = Integer.parseInt(br.readLine());
int[] a = new int[n];
int sum = 0;
StringTokenizer st = new StringTokenizer(br.readLine());
for (int i = 0; i < n; i++) {
    a[i] = Integer.parseInt(st.nextToken());
    sum += a[i];
}
for (int i = 0; i < n; i++) {
    if (sum - a[i] == a[i]) {
        System.out.println(i + 1);
        break;
    }
}
```

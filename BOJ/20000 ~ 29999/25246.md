# [25246 : Brexiting and Brentering](https://www.acmicpc.net/problem/25246)
https://www.acmicpc.net/problem/25246

## 문제
이것은 브렉시트에 관한 문제가 아닙니다. 최소한 브렉시트의 사회적, 경제적 영향에 관한 것은 아닙니다. 대신, 우리 -- 문법 정확성 감시 위원회(GCPC) -- 는 "Britain"과 "exit"이라는 단어의 조합이 제기하는 언어적 도전에만 초점을 맞추고 싶습니다. 우리는 이 구성의 모호성에 대해 매우 우려하고 있습니다. 브렉시트라는 용어를 사용할 때, 그것이 영국을 지칭하는지, 브라질을 지칭하는지, 또는 아마도 브레멘이 분데스리가를 떠나는 것을 의미하는지 전혀 명확하지 않습니다.

브렉시트뿐만이 아닙니다. 여러분은 "Megxit"(영국 왕실과 관련된)과 같은 비슷한 구성을 들어보셨을 것입니다.

미래를 내다보며, 우리는 앞으로 비슷한 재앙을 피하고 싶습니다. 이를 위해 우리는 독일에 뿌리를 둔 유럽 기관으로서 일부 표준화를 구현하고자 합니다. 우리는 '들어가는' 행위에 초점을 맞추었습니다. 만약 어떤 주체(사람, 조직, 공장 등)가 무언가에 들어가거나(또는 재진입하는 경우), 우리의 주요 언어 과학자들은 주체 이름의 마지막 모음을 찾을 것을 제안합니다 (`a`, `e`, `i`, `o`, `u`가 모음으로 간주됨). 이 마지막 모음 이후의 모든 문자를 잘라내고 대신 `ntry`라는 끝말을 붙입니다. 다음은 몇 가지 예시입니다:

- 만약 Britain이 유럽 연합에 재진입한다면, 우리는 이를 "Britaintry"라고 부를 것입니다¹.
- 만약 Canada가 NWERC 지역에 진입한다면, 그것은 "Canadantry"가 될 것입니다.
- 그리고 "Paul"이라는 이름을 가진 사람이 어딘가에 들어갈 때마다, 우리는 분명히 "Pauntry"를 갖게 됩니다.

주체의 이름이 주어졌을 때, 이 주체의 진입 행위를 어떻게 불러야 하는지 결정하세요.

<hr/>

¹우리는 이것이 일어날 것이라고 확신합니다. 브렉시트가 영국 국민들에게 그토록 큰 기쁨이었을 것이기 때문입니다 (그렇지 않다면, 그들의 정치 지도자들이 분명히 이런 조치를 취하지 않았을 것입니다). 무언가가 모두에게 그토록 큰 기쁨을 가져다주면, 여러분은 그것을 반복하고 싶어집니다. 하지만 브렉시트를 반복하기 위해서는, 먼저 Britaintry가 있어야 합니다!

<br/>

**입력**  
입력은 다음과 같이 구성됩니다:
- 문자열 $s$를 포함한 한 줄 ($1 \le |s| \le 50$), 주체의 이름입니다. 이 이름은 어떤 사람, 동물, 국가, 조직 등을 지칭할 수 있습니다.

$s$의 모든 문자는 대문자 `A`-`Z` 또는 소문자 `a`-`z`입니다. 첫 글자는 대문자나 소문자일 수 있으며, 다른 모든 글자는 소문자입니다. $s$는 최소한 하나의 소문자 모음을 포함합니다.

<br/>

**출력**  
주체의 진입 행위를 나타내는 단어를 하나만 출력하세요.

<br/>

번역 출처 : https://www.acmicpc.net/board/view/152115

## 아이디어
String, Implementation

마지막 모음 위치에서 `"ntry"`를 붙여 출력하면 된다.
```python
VOWELS = ['a', 'e', 'i', 'o', 'u']

s = input()

for i in range(len(s) - 1, -1, -1):
    if s[i] in VOWELS:
        result = s[:i+1] + "ntry"
        break

print(result)
```

# [29467 : Шифровка](https://www.acmicpc.net/problem/29467)
https://www.acmicpc.net/problem/29467

## 문제
**문제**  
Petya와 Vasya는 스파이 놀이를 좋아합니다. 

그리고 스파이 놀이에 비밀 암호가 없으면 완성됩니까!

그래서 Vasya는 최근 자신의 메시지를 암호화하는 새로운 방법을 생각해 냈습니다. 

그는 암호화 방법에 대한 정보를 비밀로 유지하고 Petya에게 항상 사용할 수 있도록 암호 해독 방법만 알려주었습니다.

해독하려면 Petya는 Vasya가 전송하는 암호화된 메시지에서 사전 순으로 최대가 되는 부분 문자열을 찾으면 됩니다.

소스 코드가 될 부분입니다.

Petya는 원하는 만큼 빨리 이 일을 할 수 없기 때문에, 그는 당신에게 이 일을 도와줄 프로그램을 써달라고 요청했어요!

<br/>

**입력**  
입력 파일의 첫 번째이자 유일한 줄에는 알파벳의 소문자로만 구성된 암호화된 메시지가 포함되어 있습니다.

메시지의 길이는 1000자를 초과할 수 없습니다.

<br/>

**출력**  
해독된 메시지를 출력합니다.

<br/>

번역 출처 : https://www.acmicpc.net/board/view/151479

## 아이디어
Brute Force, String, Implementation

메시지 길이가 작은 편이므로, 모든 부분 문자열을 생성( $O(N^2)$ )해보고 그 중 사전순으로 가장 큰 것을 찾는 방법을 사용할 수 있겠다.
```java
String encrypted = new BufferedReader(new InputStreamReader(System.in)).readLine();
String decrypted = "";
for (int i = 0; i < encrypted.length(); i++) {
    for (int j = i + 1; j <= encrypted.length(); j++) {
        // 모든 부분 문자열 탐색
        String substring = encrypted.substring(i, j);
        // 부분 문자열 최대값 찾기
        if (decrypted.compareTo(substring) < 0) {
            decrypted = substring;
        }
    }
}
System.out.println(decrypted);
```

# [11176 : In the Shower](https://www.acmicpc.net/problem/11176)
https://www.acmicpc.net/problem/11176

## 문제
**문제**  
최근 룸메이트가 당신에게 다소 소심하면서도 공격적으로 "샤워실에 빈 샴푸 통을 몇 개나 놓을 생각이야?"라는 포스트잇 메모를 남겨 놓았습니다. 소심하면서도 공격적인 포스트잇 메모는 보통 소심하면서도 공격적인 평서문 또는 의문문의 수를 나타내는 하나의 숫자 P로 시작합니다. 이후 P개의 줄 각각에 소심하면서도 공격적인 평서문 또는 의문문이 담겨 있죠. P가 1일 때는 P를 생략하며, 이때는 메모 전체가 하나의 소심하면서도 공격적인 평서문 또는 의문문이 됩니다. 최근까지 가장 흔한 유형이었죠.

샴푸 통의 개수를 세지 못하는 룸메이트에게 화가 난 당신은, 이곳 001호에서의 행복한 공존을 위해 몇 개의 샴푸 통이 있어야 하는지 궁금할 때마다 룸메이트가 실행할 수 있는 프로그램을 작성하려 합니다. 물론 (높은 확률로) 다른 룸메이트는 "빈" 샴푸 통에 대한 다른 정의를 가질 수 있으므로, 답을 출력받기 전 정의를 입력해야만 합당한 값이 나올 수 있을 것입니다.

<br/>

**입력**  
첫 줄에 테스트 케이스의 수 T가 주어집니다. 이후 T개의 테스트 케이스가 주어집니다.

각 테스트 케이스에 첫 줄에는 두 숫자가 하나의 공백으로 구분되어 주어집니다: E는 통이 비기 직전까지 샴푸를 일정량 덜어내는 동작을 반복해야 하는 횟수이며, N은 샴푸 통의 수입니다.

이후 N개의 줄 각각에 특정 통에서 샴푸를 일정량 덜어낸 횟수를 나타내는 하나의 수가 주어집니다. 만약 횟수가 E를 넘어섰다면, 병은 비었다고 간주합니다.

- 1 <= T <= 100

- 1 <= E <= 1000

- 1 <= N <= 10

<br/>

**출력**  
빈 샴푸 통의 수를 한 줄에 출력합니다.

<br/>

번역 출처 : https://www.acmicpc.net/board/view/93691

## 아이디어
Implementation

(케이스 별로) E가 넘어가는 수가 몇 번 나오는지 세어주면 된다.
```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));

int T = Integer.parseInt(br.readLine());
while (T-- > 0) {
    int emptyBottle = 0;
    StringTokenizer st = new StringTokenizer(br.readLine());
    int E = Integer.parseInt(st.nextToken());
    int N = Integer.parseInt(st.nextToken());
    while (N-- > 0) {
        if (Integer.parseInt(br.readLine()) > E) {
            emptyBottle++;
        }
    }
    bw.write(emptyBottle + "\n");
}
```
